# Icons

The FrancesHR application uses **Lucide React** icons with a custom configuration for a clean, minimal aesthetic.

## Icon Configuration

All icons in the FrancesHR application follow these specifications:

- **Library**: [Lucide React](https://lucide.dev/)
- **Stroke Width**: `1px` (thinner than default 2px for a more refined look)
- **Color**: `currentColor` (inherits from parent, typically primary color)
- **Default Size**: `16px` (1rem)

### Usage Example

```tsx
import { User } from "lucide-react";

// Default usage with 1px stroke
<User className="text-primary" strokeWidth={1} size={16} />

// Custom size
<User className="text-primary" strokeWidth={1} size={24} />
```

## Usage Guidelines

### Stroke Width

**Always use `strokeWidth={1}` for consistency:**

```tsx
// ✅ Correct
<User strokeWidth={1} size={16} className="text-primary" />

// ❌ Incorrect (uses default 2px stroke)
<User size={16} className="text-primary" />
```

### Color

**Use Tailwind color classes for consistency:**

```tsx
// Primary color (default)
<User strokeWidth={1} className="text-primary" />

// Muted/secondary
<User strokeWidth={1} className="text-muted-foreground" />

// Destructive/error
<Trash2 strokeWidth={1} className="text-destructive" />

// Success
<CheckCircle strokeWidth={1} className="text-green-600" />
```

### Size Guidelines

- **12px**: Inline with small text (captions, badges)
- **16px**: Default size for most UI elements
- **20px**: Buttons, form inputs, navigation items
- **24px**: Section headers, card headers
- **32px+**: Feature highlights, empty states

### Accessibility

- Always provide meaningful context (aria-label or sr-only text)
- Ensure sufficient color contrast (minimum 4.5:1 ratio)
- Don't rely solely on icons to convey meaning
- Use semantic HTML alongside icons

### Best Practices

- **Consistency**: Always use `strokeWidth={1}` across the application
- **Alignment**: Center icons vertically with adjacent text
- **Spacing**: Add appropriate margin/padding around icons
- **Color**: Use `currentColor` or Tailwind classes, avoid hardcoded colors
- **Size**: Match icon size to text size for inline icons
- **Loading**: Consider using skeleton loaders for async icon content

### Common Patterns

```tsx
// Button with icon
<Button>
  <Plus strokeWidth={1} size={16} />
  Add Item
</Button>

// Icon-only button
<Button variant="ghost" size="icon">
  <Settings strokeWidth={1} size={16} />
  <span className="sr-only">Settings</span>
</Button>

// Input with icon
<div className="relative">
  <Search strokeWidth={1} size={16} className="absolute left-3 top-1/2 -translate-y-1/2 text-muted-foreground" />
  <Input className="pl-10" placeholder="Search..." />
</div>
```
